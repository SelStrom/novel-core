## Добро пожаловать!

Добро пожаловать в Конструктор визуальных новелл! Этот инструмент помогает создавать профессиональные визуальные новеллы для Steam и мобильных платформ **без написания кода**. Если вы умеете писать истории и пользоваться базовыми компьютерными программами, вы сможете создать визуальную новеллу.

**Что вы сможете делать**:
- Писать разветвлённые истории с выборами игрока
- Добавлять спрайты персонажей с эмоциями
- Вставлять фоновые изображения и музыку
- Публиковать на Windows, macOS, iOS и Android

**Что вам не нужно**:
- Опыт программирования
- Знания разработки игр
- Технические навыки

Давайте начнём! 🎮

## Начало работы

### Расположение проекта Unity

Ваш проект Unity находится в папке `novel-core`. Здесь хранятся все файлы игры, скрипты и ресурсы.

```
novel-core/                 # Ваш проект Unity (корень репозитория)
├── Assets/                 # Игровые ресурсы и код
│   ├── Scripts/            # Скрипты C#
│   ├── Content/            # Контент вашей визуальной новеллы
│   └── Resources/          # Игровые ресурсы
├── Packages/               # Пакеты Unity
└── ProjectSettings/        # Настройки проекта
```

**Для разработчиков**: Попросите разработчика настроить проект Unity, если это ещё не сделано.

### Открытие конструктора

1. **Запустите Unity** (программу, которую установил для вас разработчик)
2. **Откройте ваш проект** (он должен открыться автоматически)
3. **Дождитесь загрузки Unity** (вы увидите экран загрузки с логотипом Unity)

### Понимание интерфейса

Когда Unity откроется, вы увидите несколько окон:

```
┌─────────────────────────────────────────────────────────┐
│  Панель меню (здесь находится меню NovelCore)           │
├──────────────┬──────────────────────┬───────────────────┤
│              │                      │                   │
│  Hierarchy   │   Scene View         │   Inspector       │
│  (Список     │   (Что вы видите)    │   (Свойства)      │
│  объектов)   │                      │                   │
│              │                      │                   │
├──────────────┴──────────────────────┴───────────────────┤
│  Project (Ваши файлы и ресурсы)                         │
└─────────────────────────────────────────────────────────┘
```

**Не беспокойтесь о большинстве из них!** В основном вы будете использовать меню **NovelCore**.

### Создание первого проекта

1. Нажмите **NovelCore → New Project** в панели меню
2. Введите детали проекта:
   - **Project Name**: "Моя первая визуальная новелла"
   - **Author**: Ваше имя
   - **Description**: Краткое описание вашей истории
3. Нажмите **Create**

Папка вашего проекта появится в папке **Content/Projects/**.

## Создание первой сцены

**Сцена** — это один момент или локация в вашей истории. Думайте о ней как о странице в книге или панели в комиксе.

### Шаг 1: Откройте редактор сцен

1. Нажмите **NovelCore → Scene Editor** в меню
2. Откроется новое окно с названием "Scene Editor"

### Шаг 2: Создайте новую сцену

1. Нажмите кнопку **+ New Scene** (вверху слева)
2. Заполните детали:
   - **Scene Name**: "Открывающая сцена"
   - **Scene ID**: `scene_001` (должен быть уникальным)

### Шаг 3: Добавьте фон

Каждой сцене нужно фоновое изображение.

1. **Найдите ваше изображение**:
   - Перейдите в окно **Project** внизу
   - Перейдите в **Content → Backgrounds**
   - Ваш разработчик должен был импортировать изображения сюда

2. **Перетащите**:
   - Нажмите и удерживайте фоновое изображение
   - Перетащите его в поле **Background** в редакторе сцен
   - Отпустите кнопку мыши

**Совет**: Если вы не видите свои изображения, попросите разработчика импортировать их. Поддерживаемые форматы: PNG, JPG

### Шаг 4: Добавьте фоновую музыку (опционально)

1. В окне **Project** перейдите в **Content → Audio → Music**
2. Перетащите музыкальный файл в поле **Background Music**
3. Музыка будет автоматически зацикливаться во время сцены

### Шаг 5: Сохраните сцену

1. Нажмите кнопку **Save Scene** (внизу справа)
2. Ваша сцена теперь сохранена!

## Написание диалогов

Диалог — это текст, который произносят персонажи в вашей визуальной новелле.

### Добавление реплик диалога

1. В **Scene Editor** найдите секцию **Dialogue Lines**
2. Нажмите кнопку **+ Add Line**
3. Заполните диалог:

```
┌─────────────────────────────────────────┐
│ Speaker: [Выберите персонажа или Narrator] │
│ Text: "Привет! Добро пожаловать в мою историю." │
│ Emotion: neutral                         │
└─────────────────────────────────────────┘
```

**Объяснение полей**:
- **Speaker**: Кто говорит? (Выберите из ваших персонажей или "Narrator")
- **Text**: Что они говорят (печатайте как в текстовом редакторе)
- **Emotion**: Выражение лица персонажа (happy, sad, angry и т.д.)

### Добавление нескольких реплик

Продолжайте нажимать **+ Add Line** для добавления дополнительных реплик. Они будут воспроизводиться по порядку сверху вниз.

**Пример разговора**:
```
Реплика 1: Алиса (happy): "Привет! Я Алиса."
Реплика 2: Боб (neutral): "Приятно познакомиться, Алиса."
Реплика 3: Алиса (excited): "Хочешь исследовать лес?"
```

### Редактирование диалога

- **Изменить текст**: Нажмите на поле текста и печатайте
- **Переупорядочить реплики**: Перетащите значок ☰ слева для перемещения реплик вверх/вниз
- **Удалить реплику**: Нажмите значок 🗑️ корзины справа

### Текст рассказчика

Для повествования (без говорящего персонажа):
1. Установите **Speaker** в "Narrator"
2. Текст появится по центру без портрета персонажа

## Добавление персонажей

Персонажи — это люди (или существа!) в вашей визуальной новелле.

### Шаг 1: Откройте редактор персонажей

1. Нажмите **NovelCore → Character Editor** в меню
2. Откроется новое окно с названием "Character Editor"

### Шаг 2: Создайте нового персонажа

1. Нажмите кнопку **+ New Character**
2. Заполните детали:
   - **Character Name**: "Алиса"
   - **Character ID**: `alice` (короткий, без пробелов)

### Шаг 3: Добавьте спрайты персонажа

Спрайт — это изображение персонажа. Вам нужны разные спрайты для разных эмоций.

1. **Добавьте нейтральную эмоцию** (обязательно):
   - Нажмите **+ Add Emotion**
   - Emotion Name: `neutral`
   - Перетащите изображение спрайта из папки **Content → Characters → Alice**
   - Это выражение по умолчанию

2. **Добавьте больше эмоций**:
   - Снова нажмите **+ Add Emotion**
   - Emotion Name: `happy`
   - Перетащите изображение спрайта счастливой эмоции
   - Повторите для: `sad`, `angry`, `surprised`, `worried` и т.д.

**Совет**: Вам нужна минимум одна эмоция (`neutral`). Больше эмоций = более выразительные персонажи!

### Шаг 4: Расположите персонажа в сцене

1. Вернитесь в **Scene Editor**
2. В секции **Characters** нажмите **+ Add Character**
3. Выберите вашего персонажа (например, "Алиса")
4. Установите позицию:
   - **X Position**: 0.25 = слева, 0.5 = по центру, 0.75 = справа
   - **Y Position**: Обычно 0.5 (середина)
5. Выберите **Initial Emotion** (с какой эмоции они начинают в этой сцене)

**Визуальная схема**:
```
Позиция на экране:
┌──────────────────────────────────┐
│  0.0          0.5          1.0   │  ← X Position
│   ◀           ◀            ◀      │
│  Слева      Центр        Справа  │
└──────────────────────────────────┘
```

### Шаг 5: Сохраните персонажа

Нажмите кнопку **Save Character** в редакторе персонажей.

## Создание ветвлений сюжета

Ветвления сюжета позволяют игрокам делать выборы, которые влияют на исход.

### Понимание ветвлений

```
Сцена A: "Хочешь исследовать лес?"
   ├─ Выбор 1: "Да" → Сцена B (Лесная тропа)
   └─ Выбор 2: "Нет"  → Сцена C (Остаться в городе)
```

### Добавление точки выбора

1. В **Scene Editor** прокрутите до секции **Choices**
2. Нажмите **+ Add Choice**
3. Заполните выбор:

```
┌─────────────────────────────────────────┐
│ Question: "Что ты хочешь сделать?"      │
│                                          │
│ Option 1:                                │
│   Text: "Исследовать лес"               │
│   Goes To: [Выберите сцену]             │
│                                          │
│ Option 2:                                │
│   Text: "Остаться в городе"             │
│   Goes To: [Выберите сцену]             │
│                                          │
│ [+ Add Option]                           │
└─────────────────────────────────────────┘
```

**Объяснение полей**:
- **Question**: Опциональный текст, показываемый перед выборами (например, "Что ты будешь делать?")
- **Option Text**: Текст кнопки, на которую нажимают игроки (например, "Пойти налево")
- **Goes To**: Какая сцена будет следующей (выберите из выпадающего списка)

### Создание множественных концовок

1. Создайте несколько "конечных" сцен (например, `ending_good`, `ending_bad`)
2. Добавьте выборы в ранних сценах, которые ведут к разным концовкам
3. Используйте окно **Story Flow** для визуализации ваших ветвлений

### Просмотр потока сюжета

1. Нажмите **NovelCore → Story Flow** в меню
2. Вы увидите граф, показывающий:
   - Блоки = Сцены
   - Стрелки = Связи (поток диалогов или выборы)

**Пример**:
```
┌─────────┐
│ Сцена 1 │───→ Диалог
└────┬────┘
     │ Выбор
     ├───→┌─────────┐
     │    │ Сцена 2A│──→ Концовка A
     │    └─────────┘
     │
     └───→┌─────────┐
          │ Сцена 2B│──→ Концовка B
          └─────────┘
```

**Совет**: Если вы видите красный значок предупреждения на сцене, это означает, что выбор указывает на несуществующую сцену. Создайте эту сцену или исправьте выбор!

## Добавление аудио

Аудио делает вашу визуальную новеллу более захватывающей.

### Типы аудио

1. **Фоновая музыка**: Зацикливается во время сцены (например, тема мирного города)
2. **Звуковые эффекты**: Воспроизводятся один раз для событий (например, открывающаяся дверь, шаги)
3. **Озвучка**: Голоса персонажей, произносящих диалоги (опционально)

### Добавление фоновой музыки

1. В **Scene Editor** найдите поле **Background Music**
2. Перетащите музыкальный файл из **Content → Audio → Music**
3. Музыка будет плавно появляться при начале сцены и исчезать при её окончании

**Совет**: Используйте файлы .mp3 или .ogg для лучшей совместимости

### Добавление звуковых эффектов

1. В секции **Dialogue Lines** выберите реплику диалога
2. Найдите поле **Sound Effect**
3. Перетащите звуковой файл из **Content → Audio → SFX**
4. Звук воспроизводится, когда появляется эта реплика диалога

**Примеры**:
- Звук закрывающейся двери, когда в диалоге говорится "Она вышла из комнаты."
- Звук грома, когда в диалоге говорится "Внезапно ударила молния!"
- Звук телефона, когда в диалоге говорится "Зазвонил телефон."

### Добавление озвучки

Если у вас есть актёры озвучки:
1. В секции **Dialogue Lines** выберите реплику
2. Найдите поле **Voice Clip**
3. Перетащите аудиофайл озвучки из **Content → Audio → Voice**
4. Голос воспроизводится при появлении диалога (текст автоматически продолжается после окончания голоса)

### Советы по аудио

- **Музыка**: Используйте спокойную музыку для мирных сцен, напряжённую музыку для драматических сцен
- **Звуковые эффекты**: Не злоупотребляйте! Добавляйте звуки только для важных событий
- **Озвучка**: Убедитесь, что файлы озвучки точно соответствуют тексту (тот же язык, та же эмоция)

## Предпросмотр новеллы

Всегда просматривайте свою работу перед публикацией!

### Быстрый предпросмотр

1. Нажмите **NovelCore → Preview Scene** в меню
2. Unity перейдёт в "Play Mode" (вы увидите цветной оттенок)
3. Ваша сцена будет воспроизводиться как в финальной игре:
   - Появляется фон
   - Появляются персонажи
   - Воспроизводится диалог
   - Играет музыка
4. Нажмите на экран или нажмите **Пробел** для продвижения диалога
5. Если есть выборы, нажмите на кнопки выбора

### Выход из предпросмотра

1. Нажмите кнопку **Play** (▶) вверху Unity, чтобы остановить предпросмотр
2. Вы вернулись в режим редактирования

### Тестирование ветвлений сюжета

1. Просмотрите сцену с выборами
2. Сделайте выбор (например, "Пойти налево")
3. Проверьте, что переход идёт на правильную следующую сцену
4. Остановите предпросмотр и перезапустите для тестирования других выборов

**Важно**: Любые выборы, которые вы делаете в предпросмотре, не сохраняются! Каждый предпросмотр начинается заново.

## Публикация новеллы

Когда ваша визуальная новелла готова, пришло время публиковать!

### Шаг 1: Откройте окно сборки

1. Нажмите **NovelCore → Build Pipeline** в меню
2. Откроется новое окно с названием "Build Pipeline"

### Шаг 2: Настройте параметры сборки

Заполните информацию о вашей игре:
- **Product Name**: "Моя удивительная визуальная новелла"
- **Company Name**: Ваше имя или студия
- **Version**: 1.0.0 (увеличивайте для обновлений: 1.1.0, 1.2.0 и т.д.)

### Шаг 3: Выберите платформы

Отметьте платформы, на которые вы хотите публиковать:
- ☑ **Windows** (ПК через Steam или itch.io)
- ☑ **macOS** (Mac через Steam или itch.io)
- ☑ **iOS** (iPhone/iPad через App Store)
- ☑ **Android** (Google Play Store)

**Примечание**: Сборка для iOS требует компьютер Mac. Сборка для Android требует дополнительной настройки (спросите разработчика).

### Шаг 4: Соберите вашу игру

1. Нажмите кнопку **Build All Platforms** (или **Build Windows** только для одной)
2. Дождитесь завершения сборки (это может занять 5-15 минут)
3. По завершении вы увидите сообщение об успехе

### Шаг 5: Найдите вашу сборку

Ваша собранная игра будет в:
- **Windows**: папка `Builds/Windows/` (содержит файл `.exe`)
- **macOS**: папка `Builds/macOS/` (содержит файл `.app`)
- **iOS**: папка `Builds/iOS/` (проект Xcode, нужен разработчик)
- **Android**: папка `Builds/Android/` (файл `.apk` или `.aab`)

### Шаг 6: Протестируйте вашу сборку

**Перед публикацией**:
1. Перейдите в папку сборки (например, `Builds/Windows/`)
2. Дважды кликните на исполняемый файл игры для запуска
3. Пройдите всю вашу визуальную новеллу
4. Проверьте:
   - Все сцены работают
   - Все выборы работают
   - Всё аудио воспроизводится
   - Все изображения отображаются корректно
   - Не появляются сообщения об ошибках

### Публикация в Steam

1. Создайте аккаунт **Steam Partner** (https://partner.steamgames.com)
2. Оплатите сбор за подачу приложения $100 (однократно)
3. Следуйте инструкциям по загрузке Steam (используйте папки `Builds/Windows/` и `Builds/macOS/`)
4. Отправьте на проверку

**Примечание**: Ваш разработчик может помочь с техническим процессом загрузки в Steam.

### Публикация на мобильные платформы

**iOS (App Store)**:
1. Нужен аккаунт Apple Developer ($99/год)
2. Нужен компьютер Mac с Xcode
3. Ваш разработчик должен обработать загрузку iOS сборки (требуется подпись кода)

**Android (Google Play)**:
1. Нужен аккаунт Google Play Developer ($25 однократно)
2. Загрузите файл `.aab` из `Builds/Android/`
3. Заполните список магазина (скриншоты, описание)
4. Отправьте на проверку

## Решение проблем

### Моё фоновое изображение не отображается

**Решение**:
1. Проверьте, что изображение находится в папке **Content → Backgrounds**
2. Кликните правой кнопкой на изображение → **Addressables → Mark Addressable**
3. Попробуйте перетащить его в поле Background снова

### Спрайт моего персонажа не появляется

**Решение**:
1. Убедитесь, что вы добавили хотя бы эмоцию `neutral`
2. Проверьте, что спрайт находится в папке **Content → Characters**
3. В Scene Editor убедитесь, что вы добавили персонажа в список **Characters**
4. Проверьте, что **Initial Emotion** установлена на существующую эмоцию (например, `neutral`)

### Текст диалога не отображается

**Решение**:
1. Проверьте, что вы добавили реплики диалога в **Scene Editor → Dialogue Lines**
2. Убедитесь, что поле **Text** не пустое
3. Попробуйте просмотреть сцену снова

### Кнопки выбора не появляются

**Решение**:
1. Проверьте, что вы добавили выборы в **Scene Editor → Choices**
2. Убедитесь, что каждая опция имеет заполненное поле **Text**
3. Убедитесь, что для каждой опции выбрана сцена **Goes To**
4. Убедитесь, что целевая сцена существует

### Музыка не воспроизводится

**Решение**:
1. Проверьте, что аудиофайл находится в папке **Content → Audio → Music**
2. Поддерживаемые форматы: .mp3, .ogg, .wav
3. Кликните правой кнопкой на аудио → **Addressables → Mark Addressable**
4. Попробуйте перетащить его в поле Background Music снова
5. Проверьте, что громкость вашего компьютера не выключена

### Сборка завершается с ошибкой

**Решение**:
1. Нажмите **NovelCore → Validate Project** (проверяет на ошибки)
2. Прочитайте сообщения об ошибках (они объясняют, что не так)
3. Частые проблемы:
   - Отсутствующий ресурс: Добавьте недостающий файл изображения/аудио
   - Сломанная ссылка: Исправьте выбор, указывающий на несуществующую сцену
   - Недопустимый персонаж: Проверьте, что у персонажа есть хотя бы эмоция `neutral`
4. Если не можете исправить, обратитесь за помощью к разработчику

### Режим предпросмотра завис

**Решение**:
1. Нажмите кнопку **Play** (▶) вверху для остановки предпросмотра
2. Если Unity зависает, закройте и откройте Unity заново
3. Часто сохраняйте свою работу (File → Save Project)

## Советы и лучшие практики

### Советы по написанию

**✅ ДЕЛАЙТЕ**:
- Держите диалоги короткими (2-3 предложения на реплику)
- Дайте каждому персонажу уникальный голос/личность
- Используйте эмоции в соответствии с настроением диалога
- Добавляйте текст рассказчика для описаний сцен
- Тестируйте каждое ветвление сюжета

**❌ НЕ ДЕЛАЙТЕ**:
- Не пишите стены текста (игроки будут их пропускать)
- Не забывайте добавлять выборы (линейные истории скучны)
- Не используйте слишком много звуковых эффектов (становится раздражающим)
- Не создавайте тупиковые ветви (каждый путь должен вести к концовке)

### Организация ресурсов

**Структура папок**:
```
Content/
├── Backgrounds/
│   ├── town_day.png
│   ├── town_night.png
│   └── forest.png
├── Characters/
│   ├── Alice/
│   │   ├── alice_neutral.png
│   │   ├── alice_happy.png
│   │   └── alice_sad.png
│   └── Bob/
│       └── ... (спрайты Боба)
└── Audio/
    ├── Music/
    │   ├── town_theme.mp3
    │   └── battle_theme.mp3
    └── SFX/
        ├── door_close.wav
        └── footsteps.wav
```

**Соглашения об именовании**:
- Используйте нижний регистр с подчёркиваниями: `forest_morning.png` ✅
- Избегайте пробелов: `Forest Morning.png` ❌
- Избегайте специальных символов: `forest#morning!.png` ❌

### Советы по производительности

**Для мобильных платформ**:
- Держите изображения под 2048×2048 пикселей
- Используйте .jpg для фонов (меньший размер файла)
- Используйте .png для персонажей (нужна прозрачность)
- Держите общий размер проекта под 500MB

**Для Steam**:
- Используйте высококачественные изображения (до 4096×4096)
- Используйте .ogg для музыки (лучшее качество чем .mp3)
- Нет строгого ограничения размера, но избегайте превышения 5GB

### Сохранение вашей работы

**Сохраняйте часто!**:
1. Часто нажимайте **Ctrl+S** (Windows) или **Cmd+S** (Mac)
2. Нажимайте **File → Save Project** перед закрытием Unity
3. Храните резервные копии вашей папки **Content/** (копируйте на внешний диск)

### Получение помощи

**Если вы застряли**:
1. Нажмите **NovelCore → Help** → Откроется это руководство
2. Нажмите **NovelCore → Validate Project** → Проверяет на ошибки
3. Обратитесь к разработчику за технической помощью
4. Присоединитесь к форуму сообщества (ссылка в меню Help)

## Глоссарий

**Термины, которые вы встретите**:

- **Сцена (Scene)**: Один момент или локация в вашей истории
- **Реплика диалога (Dialogue Line)**: Один фрагмент текста, произнесённый персонажем или рассказчиком
- **Спрайт персонажа (Character Sprite)**: Изображение персонажа
- **Эмоция (Emotion)**: Выражение лица персонажа (happy, sad и т.д.)
- **Выбор/Ветвление (Choice/Branch)**: Точка решения, где игроки выбирают опцию
- **Фон (Background)**: Изображение за персонажами (локация)
- **Ресурс (Asset)**: Любой файл (изображение, аудио), используемый в вашей игре
- **Предпросмотр (Preview)**: Тестирование вашей игры внутри Unity
- **Сборка (Build)**: Создание финального файла игры для игроков
- **Addressable**: Система, которая загружает ресурсы (обычно вам не нужно об этом беспокоиться!)

## Приложение: Требования к изображениям

### Фоновые изображения

- **Формат**: PNG или JPG
- **Размер**: 1920×1080 (Full HD) или 1280×720 (HD)
- **Соотношение сторон**: 16:9 (широкоформатный)
- **Размер файла**: Держите под 5MB на изображение

### Спрайты персонажей

- **Формат**: PNG (с прозрачностью)
- **Размер**: 512×1024 до 1024×2048 пикселей
- **Фон**: Прозрачный (без фона)
- **Размер файла**: Держите под 2MB на спрайт

### Аудиофайлы

**Музыка**:
- **Формат**: .mp3 или .ogg
- **Битрейт**: 128-192 kbps
- **Длительность**: Любая (будет зацикливаться)
- **Размер файла**: Держите под 10MB на трек

**Звуковые эффекты**:
- **Формат**: .wav или .ogg
- **Битрейт**: 96 kbps
- **Длительность**: Под 5 секунд
- **Размер файла**: Держите под 1MB на звук

## Приложение: Общие рабочие процессы

### Рабочий процесс 1: Добавление новой сцены

1. **NovelCore → Scene Editor**
2. **+ New Scene**
3. Перетащите фоновое изображение
4. Перетащите фоновую музыку
5. Добавьте персонажей (если есть)
6. Добавьте реплики диалога
7. Добавьте выборы (если ветвление)
8. **Save Scene**
9. **Preview** для тестирования

**Время**: 10-20 минут на сцену

### Рабочий процесс 2: Создание персонажа

1. **NovelCore → Character Editor**
2. **+ New Character**
3. Введите имя и ID
4. Добавьте спрайт эмоции `neutral`
5. Добавьте другие эмоции (happy, sad и т.д.)
6. **Save Character**

**Время**: 5-10 минут на персонажа

### Рабочий процесс 3: Обновление диалога

1. **NovelCore → Scene Editor**
2. Выберите вашу сцену
3. Найдите реплику диалога для редактирования
4. Кликните на поле текста и введите изменения
5. **Save Scene**
6. **Preview** для проверки

**Время**: 1-2 минуты на изменение

### Рабочий процесс 4: Публикация обновления

1. Внесите изменения в сцены/персонажей
2. **Save All** (File → Save Project)
3. **Validate Project** (NovelCore → Validate Project)
4. Исправьте любые ошибки
5. **NovelCore → Build Pipeline**
6. Увеличьте номер версии (1.0.0 → 1.1.0)
7. **Build All Platforms**
8. Протестируйте все сборки
9. Загрузите в Steam/App Store/Google Play

**Время**: 30-60 минут (в основном время сборки)
